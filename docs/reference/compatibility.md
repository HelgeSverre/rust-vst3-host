# Plugin compatibility

How real-world VST3 plugins fare with `vst3-host`, by capability. This is a **representative
sample on macOS** (the exercised platform), not an exhaustive or cross-platform guarantee.

Generated by the bundled harness — regenerate any time:

```bash
just compat                       # discovers installed plugins
# or test specific ones:
cargo run -p vst3-host --example compatibility_matrix \
  --features cpal-backend,process-isolation -- path/to/A.vst3 path/to/B.vst3
```

The harness is crash-safe: it **probes each plugin in an isolated process first**, and only
loads in-process (to measure the detailed capabilities) when the probe is clean — so a
plugin that crashes can't take down the run.

## What the columns mean

- **Load** — loads cleanly (`✅`), crashes the probe (`💥`, then handled only via isolation),
  times out (`⏱`), or fails to load (`❌`).
- **Params** — parameter count reported by the controller.
- **Audio** — for instruments, output peak from a held note; for effects, that it processes a
  block without error (effects fed silence stay silent).
- **State** — `save_state` → change a parameter → `load_state` restores the value.
- **MIDI-out** — the plugin exposes an event-output bus (arpeggiators, MPE, …).
- **GUI** — the plugin provides an editor view.

## Notable findings

- **WaveShell crashes on load** — its own duplicate-Obj-C-class packaging, not a host bug.
  The host stays alive because the probe runs isolated; load such plugins with
  `auto_isolate_problematic(true)` or process isolation. (See
  [process isolation](../explanation/process-isolation.md).)
- **Vital is x86_64-only**, but this host is arm64, so CFBundle can't load it. The loader now
  reports the architecture mismatch explicitly. Run the host under Rosetta, or use a
  universal/arm64 build of the plugin. (Not a library limitation.)
- **Surge XT (the synth)** state round-trip shows ❌: its state is a full patch snapshot that
  doesn't reflect an isolated normalized-parameter change the way the test expects. State
  *save/restore itself works* (Dexed, OB-Xd, TyrellN6, Valhalla all round-trip).
- **MIDI-out** is ✅ only when a plugin exposes an event-output bus (Dexed, OB-Xd). Surge XT,
  TyrellN6, and Valhalla don't emit MIDI, which is expected.
- The **State** check skips meta parameters (version/compatibility flags) and processes a
  block between set and save, so it's fair to separate-component plugins.
